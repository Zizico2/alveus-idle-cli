## Findings considered and rejected

- Introduce an enum for improve versus worsen amounts: rejected because direction
  already belongs to `ImproveStatEvent` / `WorsenStatEvent`; encoding it again in
  the amount creates redundant states and awkward matching.
- Keep `Restore` and add a separate `Decay` newtype: rejected because both carry
  the same stat-scale unit and differ only by the operation applying them. It
  would preserve the current asymmetric vocabulary instead of unifying it.
- Rename the shared quantity to `StatAmount`: reasonable in isolation, but
  rejected for this plan because stored values, full/scale constants, restore
  deltas, and decay deltas will all use the type. `Stat` is the repository-wide
  unit/value type; `AnimalStat` and `EnclosureStat` remain the axis selectors.
